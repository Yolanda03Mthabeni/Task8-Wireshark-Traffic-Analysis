## Objective

The objective of this task was to capture and analyze network traffic using Wireshark. The exercise demonstrates how network packets can be monitored, filtered, and examined to better understand network communications and identify potential security concerns.

## Tools Used

* Wireshark
* Windows 11
* Wi-Fi Network Interface

## Traffic Capture Process

A packet capture was performed using the Wi-Fi network interface. During the capture, normal internet activity was generated by browsing websites and accessing online services.

The capture was allowed to run for several minutes to collect a representative sample of network traffic.

## Packet Analysis

The captured traffic contained several protocols, including:

### TCP (Transmission Control Protocol)

TCP is a connection-oriented protocol used to establish reliable communication between devices on a network.

Observed uses:

* Secure web browsing
* Application communication
* Data transmission

### TLSv1.2 (Transport Layer Security)

TLS provides encryption for communications over the internet.

Observed uses:

* Secure HTTPS connections
* Protection of sensitive information
* Authentication of remote servers

### DNS (Domain Name System)

DNS translates human-readable domain names into IP addresses.

Observed uses:

* Domain name lookups
* Website access requests
* Network resource discovery

### SSDP (Simple Service Discovery Protocol)

SSDP is used by devices on a network to discover available services automatically.

Observed uses:

* Device discovery
* Network service advertisement

## DNS Traffic Analysis

A DNS filter was applied to isolate Domain Name System traffic.

The analysis revealed:

* DNS queries sent from the local device
* DNS responses received from DNS servers
* Resolution of domain names to IP addresses

DNS traffic is important because it represents the first step in most internet communications.

## Security Considerations

The captured traffic demonstrates how network monitoring tools can provide visibility into network communications.

Potential security benefits include:

* Identifying suspicious connections
* Detecting unauthorized communications
* Investigating network incidents
* Monitoring encrypted and unencrypted traffic

## Findings

The network capture successfully identified various protocols actively used by the system, including TCP, TLS, DNS, and SSDP.

The DNS analysis demonstrated how domain name resolution occurs between a client and DNS server.

The capture also showed encrypted TLS traffic, highlighting the importance of secure communications over modern networks.

## Conclusion

Wireshark successfully captured and analyzed network traffic on the local network. The exercise demonstrated how security analysts can monitor communications, investigate protocols, and identify network activity for troubleshooting and security purposes.
